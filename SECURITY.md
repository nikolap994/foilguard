# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| 0.4.x   | ✅ |
| < 0.4   | ❌ |

## Reporting a Vulnerability

Please **do not** open a public GitHub issue for security vulnerabilities.

Report vulnerabilities privately via GitHub's [Security Advisories](https://github.com/nikolap994/foilguard/security/advisories/new) feature, or email **nikolap994@gmail.com** with:

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (optional)

You will receive a response within 48 hours. Critical issues will be patched and released within 7 days.

## Scope

- FoilGuard Chrome extension (detection bypass, XSS, privilege escalation)
- FoilVault credential storage or encryption weaknesses
- False positive/negative manipulation that could be exploited at scale

## Out of scope

- Theoretical attacks requiring physical access to the device
- Social engineering the user
- Known Chrome extension security model limitations
